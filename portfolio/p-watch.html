parking watch

Design and development for the new parking system
UI UX

oh - ver - vyoo

Parking Watch, an innovative parking meter that uses wireless data transmission. Unlike traditional parking meters, you do not need to download or use any additional programs or worry about timeout issues. There is only one thing you have to do. Go to the watch placement device and put it on, and all the problems are solved.

how we do

Know they want

We dove into user feedbackâ€” compiled, organized, and presented findings to the UX team. Users find that existing mobile apps do not solve some problems entirely. Aside from the traditional pin-pad meter, the mobile app is convenient. However, the user needs to have a smartphone, and the existing program can only input the parking time you may need in advance. Once it times out, you need to use the program again. In the choice of duration, the general meter and mobile phone applications can only input a fixed amount that the system has already written, which indirectly causes a waste of money. My role in this group project is doing Research, Wireframe and Renovate prototype.

process

Our process is based on Design Thinking theory. We aim to incorporate the key phases of Definition, Research & Analysis and usability Test in all of our projects.

concept

We want to create a shared device, and users will no longer need any external electronic devices or applications. In order to solve the problem of timeouts, we have established time accumulation, such as refueling at gas stations, without having to worry about timeouts or paying extra time. When returning to the device, time accumulation ends.
